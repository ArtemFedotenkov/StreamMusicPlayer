# Документация по реализации программы: Stream Music Player

## 1. Назначение программы

**Stream Music Player** — локальное Windows-приложение для управления музыкой во время стримов, записи видео или работы со сценами OBS Studio.

Главная идея: пользователь создаёт музыкальные плейлисты в виде вкладок, а программа автоматически воспроизводит, останавливает или переключает музыку в зависимости от событий OBS Studio, например при смене сцены.

Программа предназначена для личного использования, без серверной части, без облака и без сложной регистрации.

---

## 2. Цели проекта

### 2.1. Основные цели

1. Создать удобный музыкальный плеер для Windows.
2. Реализовать плейлисты в виде вкладок.
3. Реализовать плавные переходы между треками.
4. Реализовать плавные переходы между разными плейлистами.
5. Подключить программу к OBS Studio через OBS WebSocket.
6. Создать редактор событий: событие OBS → действие плеера.
7. Сделать настройки подключения к OBS внутри программы.
8. Сделать локальное сохранение всех настроек, плейлистов и правил.

### 2.2. Что программа не должна делать в первой версии

1. Не нужна облачная синхронизация.
2. Не нужна регистрация пользователей.
3. Не нужен встроенный магазин музыки.
4. Не нужен стриминг музыки из Spotify, YouTube Music или Apple Music.
5. Не нужна мобильная версия.
6. Не нужен сложный многопользовательский режим.

---

## 3. Целевая платформа

### 3.1. Операционная система

Основная платформа:

- Windows 10 x64
- Windows 11 x64

### 3.2. OBS Studio

Программа должна работать с OBS Studio, где включён WebSocket Server.

Начиная с OBS Studio 28, OBS WebSocket встроен в OBS по умолчанию.

Стандартные параметры подключения:

- Host: `localhost`
- Port: `4455`
- Password: задаётся пользователем в OBS

## 4. Рекомендуемый стек технологий

## 4.1. Основной стек

Рекомендуется использовать:

- Язык: **C#**
- Платформа: **.NET 10 SDK** или актуальный LTS SDK
- UI: **WPF**
- Архитектурный паттерн: **MVVM**
- Хранение данных: **SQLite**
- ORM: **Entity Framework Core** или простой SQLite-доступ без ORM
- OBS-интеграция: библиотека для OBS WebSocket v5
- Аудио: **NAudio** для MVP или **BASS/BASS.NET** для более продвинутой аудио-логики
- Сборка: `dotnet publish`
- Установщик: Inno Setup, MSIX или обычная portable-папка

## 4.2. Почему C# + WPF

1. Хорошая совместимость с Windows.
2. Удобно делать интерфейс.
3. Можно собирать через терминал без Visual Studio.
4. Хорошо подходит для локального desktop-приложения.
5. Есть библиотеки для аудио и WebSocket.
6. Можно создать нормальный `.exe`.

## 6. Основные экраны интерфейса

## 6.1. Главное окно

Главное окно должно содержать:

1. Верхнюю панель состояния.
2. Область вкладок плейлистов.
3. Список треков выбранного плейлиста.
4. Панель управления воспроизведением.
5. Блок текущего трека.
6. Кнопки перехода в настройки и редактор событий.

### 6.1.1. Верхняя панель состояния

Показывает:

- статус подключения к OBS;
- активную сцену OBS;
- текущий плейлист;
- текущий трек;
- состояние плеера: Play / Pause / Stop / Fading / Switching.

Пример:

```text
OBS: Connected | Scene: Starting Soon | Playlist: Intro | Track: ambient_loop.mp3
```

---

## 6.2. Вкладки плейлистов

Каждый плейлист отображается как вкладка.

Функции:

1. Создать новый плейлист.
2. Переименовать плейлист.
3. Удалить плейлист.
4. Переместить плейлист влево/вправо.
5. Добавить треки в плейлист.
6. Удалить треки из плейлиста.
7. Изменить порядок треков.
8. Включить трек двойным кликом.
9. Выбрать режим воспроизведения для плейлиста.

Режимы воспроизведения:

- по порядку;
- случайно;
- повторять один трек;
- повторять весь плейлист;
- остановиться после окончания плейлиста.

---

## 6.3. Список треков

Для каждого трека отображать:

1. Название файла.
2. Исполнитель, если есть metadata.
3. Название трека, если есть metadata.
4. Длительность.
5. Путь к файлу.
6. Статус файла: найден / отсутствует.

Минимальный вариант для MVP:

```text
# | Title | Duration | Path
```

Кнопки:

1. Play
2. Pause
3. Stop
4. Previous
5. Next
6. Fade Out
7. Fade In
8. Mute

Слайдеры:

1. Громкость.
2. Позиция трека.
3. Длительность crossfade.

Индикаторы:

2. Общая длительность.
3. Текущая громкость.

## 6.5. Экран настроек OBS

Экран должен называться: **OBS Connection Settings**.

Поля:

1. Host
2. Port
3. Password
4. Enable auto-connect on startup
5. Reconnect automatically
6. Reconnect interval

Кнопки:

1. Test Connection
2. Connect
3. Disconnect
4. Refresh Scenes

После успешного подключения программа должна получить список сцен OBS.

Отображать:

- статус подключения;
- версию OBS;
- версию OBS WebSocket;
- список найденных сцен.

---

## 6.6. Редактор событий

Экран должен называться: **Event Rules Editor**.

Назначение: пользователь создаёт правила, которые связывают события OBS с действиями музыкального плеера.

Пример правила:

```text
When OBS scene changes to "Starting Soon"
Then play playlist "Intro Music"
With fade-in 3 seconds
And crossfade from current track 4 seconds
```

# 7. Система событий

## 7.1. Типы событий OBS

В MVP обязательно реализовать:

1. Смена активной сцены.

После MVP можно добавить:

1. Запуск стрима.
2. Остановка стрима.
3. Запуск записи.
4. Остановка записи.
5. Пауза записи.
6. Возобновление записи.
7. Показ источника.
8. Скрытие источника.
9. Изменение состояния источника.

---

## 7.2. Типы условий

Условие правила должно включать:

1. Тип события.
2. Название сцены или источника.
3. Дополнительный фильтр, если нужен.

Для MVP:

```text
Event Type: Current Program Scene Changed
Scene Name: <selected OBS scene>
```

---

## 7.3. Типы действий плеера

Обязательные действия:

1. Play playlist.
2. Play specific track.
3. Pause.
4. Resume.
5. Stop.
6. Fade in.
7. Fade out.
8. Crossfade to playlist.
9. Crossfade to track.
10. Set volume.
11. Mute.
12. Unmute.

Дополнительные действия:

1. Play random track from playlist.
2. Continue playlist from last position.
3. Restart playlist from beginning.
4. Stop after current track.
5. Ignore if already playing selected playlist.
6. Delay action by N seconds.

---

## 7.4. Структура правила

Каждое правило должно иметь:

1. ID
2. Название правила
3. Включено / выключено
4. Тип события OBS
5. Условие события
6. Действие плеера
7. Параметры действия
8. Приоритет
9. Задержка выполнения
10. Дата создания
11. Дата изменения

Пример модели:

```json
{
  "id": "rule-001",
  "name": "Starting Soon Music",
  "enabled": true,
  "trigger": {
    "type": "SceneChanged",
    "sceneName": "Starting Soon"
  },
  "action": {
    "type": "PlayPlaylist",
    "playlistId": "playlist-intro",
    "fadeInSeconds": 3,
    "crossfadeSeconds": 4,
    "restartIfAlreadyPlaying": false
  },
  "priority": 100,
  "delaySeconds": 0
}
```

# 8. Аудио-движок

## 8.1. Основные требования

Аудио-движок должен уметь:

1. Воспроизводить локальные аудиофайлы.
2. Ставить воспроизведение на паузу.
3. Останавливать воспроизведение.
4. Переключать треки.
5. Управлять громкостью.
6. Делать fade-in.
7. Делать fade-out.
8. Делать crossfade между двумя треками.
9. Делать crossfade между треками из разных плейлистов.
10. Отслеживать окончание трека.

---

## 8.2. Поддерживаемые форматы

Минимум:

- MP3
- WAV

Желательно:

- FLAC
- OGG
- M4A

Форматы зависят от выбранной аудио-библиотеки.

---

## 8.3. Crossfade

Crossfade — это плавный переход:

1. текущий трек постепенно затухает;
2. новый трек одновременно постепенно появляется;
3. после завершения перехода старый трек полностью останавливается.

Настройки crossfade:

1. Глобальная длительность по умолчанию.
2. Длительность для конкретного события.
3. Длительность для конкретного плейлиста.

Приоритет настроек:

```text
Event Rule Crossfade > Playlist Crossfade > Global Crossfade
```

---

## 8.4. Fade In / Fade Out

Fade In:

- трек стартует с громкости 0;
- громкость постепенно поднимается до целевой громкости.

Fade Out:

- громкость текущего трека постепенно снижается до 0;
- затем трек ставится на паузу или останавливается.

---

## 8.5. Поведение при быстром переключении сцен

Если OBS быстро переключает сцены, программа не должна ломаться.

Нужно реализовать очередь команд или отмену предыдущего перехода.

Рекомендуемая логика:

1. При новом событии отменить текущий fade/crossfade.
2. Проверить приоритет нового правила.
3. Выполнить новое действие.
4. Записать действие в лог.

## 10.3. Таблицы базы данных

### Playlists

```sql
CREATE TABLE Playlists (
    Id TEXT PRIMARY KEY,
    Name TEXT NOT NULL,
    SortOrder INTEGER NOT NULL,
    PlayMode TEXT NOT NULL,
    DefaultVolume INTEGER NOT NULL,
    DefaultCrossfadeSeconds REAL NOT NULL,
    RepeatEnabled INTEGER NOT NULL,
    ShuffleEnabled INTEGER NOT NULL,
    CreatedAt TEXT NOT NULL,
    UpdatedAt TEXT NOT NULL
);
```

### Tracks

```sql
CREATE TABLE Tracks (
    Id TEXT PRIMARY KEY,
    PlaylistId TEXT NOT NULL,
    FilePath TEXT NOT NULL,
    DisplayTitle TEXT NOT NULL,
    Artist TEXT,
    DurationSeconds REAL,
    SortOrder INTEGER NOT NULL,
    Enabled INTEGER NOT NULL,
    LastKnownExists INTEGER NOT NULL,
    CreatedAt TEXT NOT NULL,
    UpdatedAt TEXT NOT NULL,
    FOREIGN KEY (PlaylistId) REFERENCES Playlists(Id)
);
```

### EventRules

```sql
CREATE TABLE EventRules (
    Id TEXT PRIMARY KEY,
    Name TEXT NOT NULL,
    Enabled INTEGER NOT NULL,
    TriggerType TEXT NOT NULL,
    TriggerSceneName TEXT,
    ActionType TEXT NOT NULL,
    ActionJson TEXT NOT NULL,
    Priority INTEGER NOT NULL,
    DelaySeconds REAL NOT NULL,
    CreatedAt TEXT NOT NULL,
    UpdatedAt TEXT NOT NULL
);
```

### AppSettings

```sql
CREATE TABLE AppSettings (
    Key TEXT PRIMARY KEY,
    Value TEXT NOT NULL
);
```

# 13. Ошибки и поведение программы

## 13.1. OBS недоступен

Если OBS не запущен или WebSocket выключен:

1. программа должна продолжать работать как обычный плеер;
2. показать статус `OBS Disconnected`;
3. попытаться переподключиться, если включён auto-reconnect.

---

## 13.2. Файл трека не найден

Если файл отсутствует:

1. пометить трек как missing;
2. пропустить его при воспроизведении;
3. показать предупреждение в интерфейсе;
4. записать ошибку в лог.

---

## 13.3. Несколько правил на одну сцену

Если несколько правил подходят под событие:

1. выполнить правило с самым высоким приоритетом;
2. если приоритет одинаковый — выполнить первое по порядку;
3. позже можно добавить режим выполнения нескольких правил.

# 17. Критерии готовности MVP

MVP считается готовым, если:

1. Программа запускается на Windows.
2. Можно создать минимум два плейлиста.
3. Можно добавить музыку в каждый плейлист.
4. Можно включить трек вручную.
5. Можно переключить трек вручную.
6. Crossfade работает между треками одного плейлиста.
7. Crossfade работает между разными плейлистами.
8. Программа подключается к OBS.
9. Программа получает список сцен OBS.
10. При смене сцены OBS выполняется выбранное действие.
11. Все настройки сохраняются после перезапуска.
12. Ошибки пишутся в лог.

# 19. Пример пользовательского сценария

1. Пользователь запускает OBS Studio.
2. Пользователь запускает Stream Music Player.
3. Программа автоматически подключается к OBS.
4. Пользователь создаёт плейлист `Starting Soon`.
5. Пользователь добавляет туда музыку.
6. Пользователь создаёт плейлист `Gameplay`.
7. Пользователь добавляет туда другую музыку.
8. Пользователь открывает Event Rules.
9. Создаёт правило:

```text
Scene: Starting Soon
Action: Play Playlist
Playlist: Starting Soon
Fade In: 3 sec
Crossfade: 4 sec
```

10. Создаёт второе правило:

```text
Scene: Gameplay
Action: Crossfade To Playlist
Playlist: Gameplay
Crossfade: 5 sec
```

11. В OBS пользователь переключается на сцену `Starting Soon`.
12. Плеер включает музыку из нужного плейлиста.
13. Пользователь переключается на `Gameplay`.
14. Плеер плавно переводит музыку на другой плейлист.

# 20. Инструкция для ИИ-разработчика

ИИ-разработчик должен реализовывать проект поэтапно.

Запрещается сразу писать весь проект одним большим файлом.

Нужно делать так:

1. Сначала создать структуру проекта.
2. Затем создать базовый UI.
3. Затем добавить модели данных.
4. Затем добавить SQLite.
5. Затем добавить аудио.
6. Затем добавить OBS.
7. Затем добавить правила.
8. Затем добавить сборку.

Каждый этап должен быть проверяемым.

После каждого этапа программа должна запускаться.

# 22. Итоговая формулировка проекта

Нужно создать Windows-приложение на C# WPF: локальный музыкальный плеер с плейлистами-вкладками, плавными переходами между треками и интеграцией с OBS Studio через OBS WebSocket. Пользователь должен иметь возможность подключиться к OBS, получить список сцен, создать правила событий и назначить действия плеера на смену сцен. Все данные должны храниться локально в SQLite. Первая версия должна быть простой, стабильной и пригодной для личного использования.
